Programming Languages Homework 2

发表于 2021-02-14 更新于 2021-02-15 分类于 学习 本文字数： 1.4k 阅读时长 ≈ 5 分钟

作业2中使用了在这部分课程中介绍的很多有用的概念。

  • 利用编程语言中自带的基本类型创建的复合类型可以分为三类：Each-of：相当于一个类，每个成员变量都需要赋值；One-of：我觉得相当于枚举类型；Self-reference：在实现链表、树等数据结构时需要对类型自身进行引用。

  • 通过Records可以可以创建Each-of类型。

  • 语法糖（Syntactic Sugar）的作用：通过语法糖可以简化编程语言中实现某一功能所需的代码。

  • 介绍了等号的第三种（第一次课程中介绍过：Value Binding和Function Binding）作用：Datatype Binding。

  • case表达式：case表达式的作用非常强大，通过case表达式，可以实现模式匹匹配（Pattern-Matching），模式匹配是这一次课程的重点。

  • 尾递归（Tail Recursion）和累加器（Accumulator）：在递归的过程中，由于需要使用下一状态的值，需要存储大量递归函数的中间状态，大量的递归会造成栈的溢出。尾递归就是通过把递归函数的结果作为参数传给下一次调用，从而节省了栈的开销。需要注意不是所有递归都能改成尾递归，并且不是所有尾递归都能节省栈的开销。

Problem 1

1-a

Write a function remove_option, which takes a string and a string list. Return NONE if the string is not in the list, else return SOME xs where xs is identical to the argument list except the string is not in it. You may assume the string is in the list at most once. Use same_string, provided to you, to compare strings. Sample solution is around 8 lines.

1-b

Write a function all_substitutions1, which takes a string list list (a list of list of strings, the substitutions) and a string s and returns a string list. The result has all the strings that are in some list in substitutions that also has s, but s itself should not be in the result.

1-c

Write a function all_substitutions2, which is like all_substitutions1 except it uses a tail-recursive local helper function.

1-d

Write a function similar_names, which takes a string list list of substitutions (as in parts (b) and (c)) and a full name of type {first:string, middle:string, last:string} and returns a list of full names (type {first:string, middle:string, last:string} list). The result is all the full names you can produce by substituting for the rst name (and only the rst name) using substitutions and parts (b) or (c). The answer should begin with the original name (then have 0 or more other names).

Problem 2

2-a

  1. Write a function card_color, which takes a card and returns its color (spades and clubs are black, diamonds and hearts are red). Note: One case-expression is enough.

2-b

Write a function card_value, which takes a card and returns its value (numbered cards have their number as the value, aces are 11, everything else is 10). Note: One case-expression is enough.

2-c

Write a function remove_card, which takes a list of cards cs, a card c, and an exception e. It returns a list that has all the elements of cs except c. If c is in the list more than once, remove only the rst one. If c is not in the list, raise the exception e. You can compare cards with =.

2-d

Write a function all_same_color, which takes a list of cards and returns true if all the cards in the list are the same color. Hint: An elegant solution is very similar to one of the functions using nested pattern-matching in the lectures.

2-e

Write a function sum_cards, which takes a list of cards and returns the sum of their values. Use a locally defined helper function that is tail recursive.

2-f

Write a function score, which takes a card list (the held-cards) and an int (the goal) and computes the score as described above.

2-g

Write a function officiate, which "runs a game." It takes a card list (the card-list) a move list (what the player "does" at each point), and an int (the goal) and returns the score at the end of the game after processing (some or all of) the moves in the move list in order. Use a locally defined recursive helper function that takes several arguments that together represent the current state of the game. As described above:

  • The game starts with the held-cards being the empty list.
  • The game ends if there are no more moves. (The player chose to stop since the move list is empty.)
  • If the player discards some card c, play continues (i.e., make a recursive call) with the held-cards not having c and the card-list unchanged. If c is not in the held-cards, raise the IllegalMove exception.
  • If the player draws and the card-list is (already) empty, the game is over. Else if drawing causes the sum of the held-cards to exceed the goal, the game is over (after drawing). Else play continues with a larger held-cards and a smaller card-list.
