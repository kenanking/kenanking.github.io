IDL基础语法

一、IDL两种工作模式

IDL存在两种基本的工作模式：交互模式（interactive mode）和编译模式（compiled mode）。前者通过在IDL命令行中输入指令执行命令并得到及时反馈，适用于程序原型设计和交互式分析。后者用于编写指令集来开发较复杂的数据分析和可视化应用。

在IDL中，基本的程序单元为过程（procedures）和函数（functions）。

二、变量

IDL中使用变量不需要申明，类型也不是固定的，在任何时候都可以直接指定新的变量。

数据类型

IDL支持大量数据类型，主要有：

byte 8 0 to 255
int 16 -32767 to 32767
uint 16 0 to 65535
long 32 \(-2^{31}\) to \(2^{31}-1\)
ulong 32 0 to \(2^{32}-1\)
long64 64 \(-2^{63}\) to \(2^{63}-1\)
ulong64 64 0 to \(2^{64}-1\)
float 32 \(-10^{38}\) to \(10^{38}\)

string 字符串（0 - 32767个字符）

objref 对象结构的引用

变量命名

变量名以字母开头，可以包含字母数字和下划线，在IDL中字母是不区分大小写的。IDL中的保留字不能作为变量名，函数名也最好不要作为变量名。

三、数组

IDL中可以由任何数据类型组成一至八维的数组。在IDL中，可以使用更简洁的语法处理数组、进行数组运算，而非使用循环，这些数组运算通常也是最优化的、执行速度比较快。
