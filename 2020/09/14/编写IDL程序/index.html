编写IDL程序

一、编写与编译IDL程序

IDL程序包含两类：过程（procedures）和函数（functions）。一个过程中通常封装了几个不同的相关操作（比如打开读取文件、显示并缩放图片等等），而一个函数只实现一个功能。

过程（procedures）

当有多个变量传递给程序或者从程序返回时，通常使用过程。一个过程通常具有一个输入参数（也可以没有输入参数），具有一个或多个输出参数。

例如，如果想以某种格式从文件中读取数据数组，则可以编写一个过程。该过程输入参数为文件名，输出参数为文件内容和大小。 在该过程中，相关操作有检查文件是否存在，然后将其打开并以正确的格式读取内容，最后关闭文件。 调用方式可能如下所示：

手动编译

过程和函数可以手动通过.compile命令进行编译：

一旦手动编译了过程或函数，则该过程或函数将驻留在内存中，以供IDL会话的其他部分使用而无需重新编译。

如果修改并重新编译了过程或函数的源代码，则新的编译版本将在内存中处于活动状态，原来的版本将丢失。

此外，手动编译源文件不会重新编译在中调用的任何过程或函数（除非它们包含在源文件本身中）。

自动编译

IDL在运行时会根据需要，自动编译过程和函数。

如果IDL遇到对先前未在当前会话中编译的过程或函数的调用，它将在当前目录和IDL搜索路径中的所有目录中搜索与该过程或函数调用同名的源文件。

但是如果一个源文件中，存在多个过程或函数，如果要顺利进行自动编译，这些过程和函数之间的顺序是非常重要的。当IDL自动编译源文件时，调用的每个过程或函数都必须满足以下条件之一：

  1. 在当前源文件中已先对其进行编译
  2. 在当前IDL会话中已先对其进行编译
  3. 可以在IDL路径中找到它并自动对其进行编译

二、控制语句

IDL中的控制语句允许指定语句执行的顺序。下表为IDL的控制语句：

语句 作用
if 如果条件成立，执行下面的语句
case 从一系列情况中，选择一个执行
for 对指定的次数，执行语句的循环
while 条件为真时，执行语句的循环
repeat 重复语句的循环，直到条件为真
return 将控制权返回给调用函数或过程
goto 前往一个标签
switch 跳转到一系列情况下的某一个情况
break 跳出循环、case语句或switch语句
continue 继续执行循环的下一次迭代
